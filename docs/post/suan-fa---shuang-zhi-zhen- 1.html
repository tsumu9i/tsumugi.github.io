移动零：给定一个数组_ nums，编写一个函数将所有 0 移动到数组的末尾，同时保持非零元素的相对顺序。

给定两个指针，让一个指针在前面探路，可以叫做快指针，而慢指针依赖于快指针的状态进行动作

从代码上可以看到，在i这个快指针往前探路(寻找一个新的非0元素的同时，找到第一个0元素，把j放在第一个0元素的位置占位)的时候，j这个慢指针实际上是没有动作的，只有满足了nums[i] != 0 这一条件后才会进行慢指针的动作，而j++这个操作实际是在i++之前，这样就能保证j先一步到达下一个位置。
如此循环，可以确保j必定会到达第一个0元素的位置
而到达0元素的时候，(假设此时二者都为n) 由于不满足j++的条件 j不变动 i变为n+1，从而产生了差值，保证i不等于j的条件满足，从而可以交换
此时i继续去寻找一个新的非0元素，(相邻很多0，或者没有相邻的0，情况相同)，找到了以后i就可以和j占据的位置的0元素交换位置，然后非0元素被挪动到前面去。
（交换一次之后）此时i和j继续变动，j其实只是把位置的索引往后挪了一位，i寻找新的非0元素，继续进行一次交换，这样就可以保证相对位置不变，而0全部到后面去了

删除有序数组中的重复项/移除元素

处理数据的方式：进行覆盖或者交换均可
由于不要求K元素之后的元素
还是运用占位的思想，让j这个慢指针占需要的位置
在26题的体现中就是j占第一个有重复元素的首位，i往前寻找第一个与j位置不同的元素，让j的第2位被替换成新的不重复元素进而完成了一次操作，j可以去下一个需求位置(有重复元素)占位，再让起初j的第3位变成更大的不重复元素。
在27题中就是j占取第一个需要删除的元素，i往前寻找第一个与j不同的元素(不用删除的)，把j这个元素进行替换，从而j可以去下一个需求位置占位(需要删除的元素val)，再让新的位置变成更大的不重复元素。

横向对比两题可以发现，j的本质作用是占位，i则是寻找，两题的差别主要在于怎么对于j所占的位置进行处理
26就是对j的下一个位置进行处理，因为要保留第一个重复(即第一个满足需求的)元素
27就是对j的位置进行处理，因为不需要寻找到的那一个(满足需求)的元素
总的来说，双指针还是相当复杂的，这种简单题目也值得我这个初学者好好思考。
